# Лабораторная работа №1
**Тема**: Формулирование требований к программной системе.
**Цель работы**: Научиться анализировать поставленную задачу, формулировать функциональные и нефункциональные требования к проектируемой системе.
## Перечень заинтересованных лиц (стейкхолдеров)

| Именование                             | Описание                                                                                                                                                      |
| :------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Преподаватели биологии                 | формируют тесты, управляют базой вопросов, назначают тестирование студентам и анализируют результаты для объективной оценки знаний                            |
| Студенты биологи                       | проходят тестирование, получают результаты и рекомендации по темам, в которых были допущены ошибки, могут проходить дополнительные тесты для улучшения знаний |
| Администрация учебной программы        | заинтересована в повышении качества образовательного процесса, цифровизации и наличии статистики по успеваемости и результатам тестирования                   |
| Научный руководитель (спонсор проекта) | согласует ключевые решения по реализации, контролирует соответствие проекту ВКР и принимает результат                                                         |

## Перечень функциональных требований
- Система должна поддерживать создание и использование тестов, состоящих из набора вопросов.
- Система должна поддерживать различные типы тестовых заданий, включая:одиночный выбор, множественный выбор,открытый ответ и сопоставление. 
- Система должна обеспечивать автоматическую проверку результатов тестирования по заданным преподавателям параметрам.
- Система должна реализовывать адаптивное обучение: если студент допустил ошибку, система автоматически формирует дополнительный тест по теме ошибки.
- Система должна вести статистику ошибок и формировать индивидуальные рекомендации студенту.
- Система должна обеспечивать отдельный веб-интерфейс для студента и преподавателя.
- Преподаватель должен иметь возможность формировать тесты (создание, редактирование, удаление).
- Преподаватель должен иметь возможность просматривать результаты студентов после прохождения теста.
- Студент должен иметь возможность проходить тестирование в веб-интерфейсе.

## Перечень сделанных предположений
- Пользователь системы проходит авторизацию и получает доступ к функциям согласно своей роли.
- Адаптивный тест формируется на основе “темы”, к которой относится вопрос — значит у каждого вопроса должна быть задана тема/раздел.
- В проект НЕ входит интеграция с LMS.
- Автоматическая проверка применяется только к тем типам заданий, которые можно формально проверить без участия преподавателя.
- Система является веб-приложением (фронтэнд бэкэнд) и работает через браузер, без оптимизации под мобильные устройства. 
- База данных тестовых заданий формируется преподавателем вручную внутри системы и посредством экспорта .csv и .xls файлов. 
## Перечень нефункциональных требований
- Производительность - система должна формировать результаты тестирования и отображать итоговую оценку студенту не более чем за 2 секунды (при отсутствии вопросов, в которых необходима проверка преподавателем).
- Надёжность и корректность данных - система должна гарантировать сохранение результатов тестирования в базе данных без потери данных при штатных сценариях работы. Также должна быть предусмотрена обработка некорректного ввода (ошибочные/пустые ответы, неверные форматы).
- Безопасность и разграничение доступа - система должна предоставлять доступ к функциям по ролям: студент видит только свои тесты и результаты,  преподаватель видит результаты студентов и управляет тестами.
- Удобство использования - интерфейс должен быть интуитивно понятным и не требовать дополнительного обучения, так как система используется в учебной среде и должна быстро внедряться.
- Документированность - система должна сопровождаться пользовательской и технической документацией (архитектура, БД), так как проект защищается как ВКР.
