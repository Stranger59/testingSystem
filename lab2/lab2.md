# Лабораторная работа №2
**Тема**: Использование нотации C4 model для проектирования архитектуры программной системы
**Цель работы**: Получить опыт использования графической нотации для фиксации архитектурных решений.
## Диаграмма системного контекста

<img width="811" height="742" alt="Диаграмма системного контекста drawio" src="https://github.com/user-attachments/assets/c645bc02-a0b0-4fe9-b235-854b0bf3e989" />

- Студент — проходит тесты, получает результаты и рекомендации.
- Преподаватель — создаёт/редактирует вопросы и тесты, смотрит результаты.
- Система тестирования студентов-биологов — проводит тестирование, сохраняет и анализирует результаты.
- Система сбора данных — принимает итоговые результаты тестирования для последующей обработки/хранения.
## Диаграмма контейнеров
Выбран базовый архитектурный стиль: слоистая веб-архитектура: Frontend → Backend API → База данных. Самый подходящий и понятный вариант для веб-системы тестирования, обеспечивает разделение ответственности и значительно упрощает разработку.

<img width="768" height="781" alt="Диаграмма контейнеров drawio" src="https://github.com/user-attachments/assets/e5537b6c-0476-405f-bf99-82b79fecaa06" />

- Веб-клиент — пользовательский интерфейс для студента и преподавателя.
- Бэкенд API (FastAPI) — точка входа для веб-клиента, реализует бизнес-логику, авторизацию, управление тестами и результатами.
- Сервис адаптивного обучения — формирует дополнительные тесты и рекомендации на основе ошибок.
- База данных (PostgreSQL) — хранит пользователей, вопросы, тесты, попытки и результаты.
## Диаграмма компонентов
Для диаграммы компонентов был выбран Бэкенд-апи контейнер.
<img width="922" height="932" alt="Диаграмма компонентов drawio" src="https://github.com/user-attachments/assets/acd53cef-d25a-49bb-a52c-cd2fbaa9f189" />

- Контроллер авторизации — логин/сессии/роли пользователей.
- Контроллер тестирования — CRUD вопросов, тем и типов вопросов.
- Контроллер управления тестами — сборка тестов из вопросов, настройки (темы, количество, сложность).
- Модуль автопроверки — автоматическая проверка ответов и подсчёт баллов.
- Модуль аналитики и результатов — сохранение результатов, статистика ошибок, отчёты для преподавателя.
- Сервис адаптивного обучения — вызывается для формирования доп. теста и рекомендаций.

<img width="422" height="812" alt="диаграмма компонентов для второго контейнера drawio" src="https://github.com/user-attachments/assets/18ca8220-94c6-49b4-ac7c-e606657d74db" />

Для дополнительной диаграммы компонентов был выбран контейнер - Сервис адаптивного обучения.
- Модуль анализа ошибок — определяет проблемные темы и правила подбора доп. заданий.
- Сервис создания теста — формирует дополнительный тест по “проблемным” темам.
- База данных — источник вопросов и место хранения результатов/рекомендаций (при необходимости).
