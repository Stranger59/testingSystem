# Документация по API
## 1. Принятые проектные решения при проектировании API
### Решение 1. Ресурсно-ориентированная структура URL
В API используются имена ресурсов во множественном числе: `/topics`, `/tests`, `/attempts`, `/recommendations`.

### Решение 2. Разделение операций по HTTP-методам
Для чтения данных используются `GET`, для создания — `POST`, для изменения — `PUT`, для удаления — `DELETE`. 

### Решение 3. Явное разделение предметных сценариев
Операции предметной области вынесены в отдельные endpoint'ы: отправка ответа (`POST /attempts/{attempt_id}/answers`), завершение попытки (`POST /attempts/{attempt_id}/finish`), генерация адаптивного теста (`POST /tests/generate-adaptive`). 

### Решение 4. Корректные коды HTTP-ответов
Используются коды:
- `200 OK` — успешное чтение или изменение;
- `201 Created` — успешное создание ресурса;
- `204 No Content` — успешное удаление;
- `400 Bad Request` — неверные входные данные;
- `404 Not Found` — ресурс не найден;
- `409 Conflict` — операция противоречит состоянию данных.

### Решение 5. Валидация входных данных через Pydantic
Для входных и выходных моделей используются схемы Pydantic. Это позволяет валидировать поля, автоматически документировать API и уменьшать количество.

### Решение 6. Сокрытие корректных ответов в endpoint получения теста
`GET /tests/{id}` возвращает студенту формулировки вопросов и варианты ответов, но не возвращает поле `is_correct`. Это предотвращает утечку правильных ответов через клиентский интерфейс.

### Решение 7. Запрет изменения завершённой попытки
После вызова `POST /attempts/{attempt_id}/finish` попытка считается завершённой, и отправка новых ответов запрещена. 

### Решение 8. Использование Swagger как части API-документации
Так как проект реализован на FastAPI, интерфейс автоматически публикуется через Swagger UI (`/docs`). 
## Описание API
### 1. Список реализованных endpoint'ов

| № | Метод | Endpoint | Назначение |
|---|---|---|---|
| 1 | GET | `/topics` | Получить список тем |
| 2 | POST | `/topics` | Создать новую тему |
| 3 | PUT | `/topics/{topic_id}` | Обновить тему |
| 4 | DELETE | `/topics/{topic_id}` | Удалить тему |
| 5 | GET | `/tests/{test_id}` | Получить тест с вопросами |
| 6 | POST | `/attempts` | Создать попытку прохождения теста |
| 7 | GET | `/attempts/{attempt_id}` | Получить информацию о попытке |
| 8 | POST | `/attempts/{attempt_id}/answers` | Сохранить ответ на вопрос |
| 9 | POST | `/attempts/{attempt_id}/finish` | Завершить попытку и вычислить результат |
| 10 | POST | `/tests/generate-adaptive` | Сгенерировать адаптивный тест |
| 11 | GET | `/recommendations/student/{student_id}` | Получить рекомендации студента |

### 2. Краткое описание контрактов API

#### 2.1. Получение списка тем
**GET** `/topics`

Ответ: массив тем.

#### 2.2. Создание темы
**POST** `/topics`

Пример тела запроса:

```json
{
  "title": "Микробиология",
  "description": "Основы микробиологии для студентов-биологов"
}
```

#### 2.3. Обновление темы
**PUT** `/topics/{topic_id}`

Пример тела запроса:

```json
{
  "title": "Обновлённая микробиология",
  "description": "Новая формулировка описания"
}
```

#### 2.4. Удаление темы
**DELETE** `/topics/{topic_id}`

#### 2.5. Получение теста
**GET** `/tests/{test_id}`

Ответ содержит:
- сведения о тесте;
- список вопросов;
- варианты ответов без признака правильности.

#### 2.6. Создание попытки
**POST** `/attempts`

```json
{
  "student_id": 2,
  "test_id": 1
}
```

#### 2.7. Получение попытки
**GET** `/attempts/{attempt_id}`

#### 2.8. Отправка ответа
**POST** `/attempts/{attempt_id}/answers`

Для вопроса с выбором вариантов:

```json
{
  "question_id": 1,
  "selected_option_ids": [1]
}
```

Для текстового вопроса:

```json
{
  "question_id": 3,
  "text_answer": "Цитоплазма"
}
```

#### 2.9. Завершение попытки
**POST** `/attempts/{attempt_id}/finish`

Ответ содержит:
- набранный балл;
- максимальный балл;
- процент выполнения;
- текст рекомендации.

#### 2.10. Генерация адаптивного теста
**POST** `/tests/generate-adaptive`

```json
{
  "student_id": 2,
  "topic_id": 1,
  "based_on_attempt_id": 1,
  "created_by": 1,
  "question_count": 4
}
```

#### 2.11. Получение рекомендаций
**GET** `/recommendations/student/{student_id}`
